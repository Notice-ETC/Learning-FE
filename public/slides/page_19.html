Best Practices and Troubleshooting

Best Practices

Naming

Use descriptive names, follow camelCase, be consistent

Error Handling

Always use try-catch, return proper status codes

Validation

Validate input data, check required fields

Code Organization

Keep functions small, separate concerns

Troubleshooting Guide

Common Errors

  • "Cannot find module" → Run npm install
  • "Port already in use" → Change PORT
  • "MongoDB connection failed" → Check if running

Debugging Tips

  • Read error messages carefully
  • Use console.log() to debug
  • Check if MongoDB is running
